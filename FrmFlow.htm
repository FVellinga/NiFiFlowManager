The initialisation form

This application is a WindowForms tray application. Only one instance of the application runs. To close the application, you have to use the 'Close' menu option of the tray icon. The tray are the incons found at the rigth bottom of the windows taskbar. You can make the application invisible as a whole, or you can make this initialisation form or window invisible.

 

The following settings must be initialised.

NiFi local config The local Apache NiFi installation configuration directory where the current flow file flows.xml.gz is stored. This directory is defined at installation time, when you install Apache NiFi on your local machine.
NiFi local flow directory The directory where all the single data flows, or other flows are stored. When flow development is under source control, using SourceTree or Githbub, then this is regarded as your 'clone' directory. It is at your local file system. Every data-flow is implemented as an incapsulation of one process group and is stored in a separate sub-directory.
NiFi local backup flow directory When you replace the current flow, flows.xml.gz, at the config directory with an other flow from the local flow directy, called a PUSH operation, then the current flow is backupped into a sub-directory at this location
Common flow The default directory where the common flow is stored. This is a sub-directory under the NiFI local flow directory. When you are merging flows, you are able to change this. The common flow contains parameter setting and other shared information used by the other flows.
Merged flow The default directory where the merged flow is stored. This is a sub-directory under the NiFI local flow directory. When you are merging flows, you are able to change this. The merged flow contains extract from the common flow and the selected other flows.
Editor alias You can use a prefered editor, to edit a .xml file. You cannot edit the content of the compressed file itself. The alias name is displayed on a button at the user interface.
Editer pathThe path to your preferred editor.
Grid The merged flow contains all the extracted/imported process groups. This is a column number telling how many are displayed per row on the canvas.

The following auto-start options exists:

auto populate Populate the node tree with the above described directories at start-up.
auto collapse Collapes the node tree after being populated with values at start-up.
auto render Render (tag coloring) the XML file when you view a XML file. For large files (10000 lines and up), this is slow. Only works when the view mode is RichtTextBox control.
auto close Close, makes invisible, this initialisation form at start-up. There is a delay of 1 second, before this happens.
view mode Default viewing is done in a so called RichTextBox control. When you select this option, it is done in the Webbrowser control. Webbrowser renders automatically but fails displaying the file content when the XML file isnvalid. RichTextBox control always displays, but color rendering can be wrong when your XML file is invaled. Well, the flow.xml.gz file should be always correct.

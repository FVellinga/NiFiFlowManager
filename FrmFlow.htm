The flow form

Here you do all the data flow manipulation. The PULLING, PUSHING and selection of flows that are part in the merging process. At the left you have a tree view, at the right the view area, at the bottom a number of buttons.  The tree view has a context menu, that is a variable, depending on which node you have selected. The same applies to the buttons.

 

The tree view context menu has the following menu options:

Refresh A refresh of the node tree. This is a re-read of the file system. All directories and content, only *.xml and *.gz, are shown.
Refresh flow A refresh of the node tree for the directories that are used by the application. Only available at the root node.
Open location folder Open windows explorer for the select directory.
Delete Delete a directory or file. No cascading.
Merge Only available for the flow directory. This selects one particular flow that is part of the merging process. You have to select the flow.xml.gz file to get this option.
Merge all Only available for the flow directory. This selects all flows for the merging process. You have to select the flow.xml.gz file to get this option.

 

The following buttons exists:

View View the XML content of a *.xml or *.gz file.
Notepad Use Notepad to view the content of a *.xml file only. This gives write capabilities.
<preferred> Use your preferred editor to view the content of a *.xml file only. This gives write capabilities.
Push Copy a data flow to the config flow. NiFi is shut down automatically. Re-start has to be done manually.
Pull Copy a config flow to a data flow. New one or existing one.
Merge Only available for the flow directory. This selects one particular flow that is part of the merging process.
Shutdown Shut down NiFi.

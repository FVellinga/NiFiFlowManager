The initialization form

This application is a C# Window Forms tray application. Only one instance of the application runs. To close the application, you have to use the 'Close' menu option of the tray icon. The tray are the icons found at the right bottom of the windows taskbar. You can make the application invisible as a whole, or you can make this initialization window invisible. The application only lists *.xml and compressed *.gz files.

Remark: This window has a Help Menu Item. The others do not. Use the F1 keyboard button instead..

 

The following settings must be specified.

NiFi config directory The local Apache NiFi installation configuration directory where the config flow file flows.xml.gz is stored. This directory is defined at installation time, when you install Apache NiFi on your local machine. The config flow is the flow that is run/execute by NiFi.
NiFi flow directory The directory where all flows are stored. (The common flow, the merged flow and the data flows.) When flow development is under source control, using SourceTree or GithHub, then this is regarded as your 'clone' directory. It is at your local file system. Every data flow is implemented as an encapsulation of one process group and is stored in a separate sub-directory.
NiFi backup directory When you replace the config flow, flows.xml.gz, at the NiFi config directory with an other flow from the NiFi flow directory, called a PUSH operation, then the config flow is back-upped into a sub-directory at this location.
Common flow The default directory where the common flow is stored. This is a sub-directory under the NiFi flow directory. When you are merging flows, you are able to change this. The common flow contains parameter setting and other shared information used by the data flows. It must not contain a process group.
Merged flow The default directory where the merged flow is stored. This is a sub-directory under the NiFi flow directory. When you are merging flows, you are able to change this. The merged flow contains extract from the common flow and the selected data flows.
Editor alias You can use a preferred editor to edit a *.xml file. You cannot edit the content of a compressed file itself. The alias name is displayed on a button at the user interface.
Editor pathThe path to your preferred editor.
Grid The merged flow contains all the extracted/imported process groups. This is a column number telling how many are displayed per row on the canvas. Once specified, the merged flow get x/y coordinates. Changes here are not reflected in the merged flow, unless you generate again.

 

The following auto-start options exists:

auto populate Populate the node tree with the above described directories at start-up.
auto collapse Collapse the node tree after being populated with values at start-up.
auto render Render (tag coloring) the XML file when you view a XML file. For large files (10000 lines and up), this is slow. Only works when the view mode is RichTextBox control.
auto close Close, makes invisible, this initialization form at start-up. There is a delay of 1 second, before this happens.
view mode Default viewing is done in a so called RichTextBox control. When you select this option, it is done in the Webbrowser control. Webbrowser renders automatically but fails displaying the file content when the XML file is invalid. RichTextBox control always displays, but color rendering can be wrong when your XML file is invalid. Well, the flow.xml.gz file should be always correct.
